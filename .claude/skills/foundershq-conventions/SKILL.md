# FoundersHQ conventions

This skill is the project's load-bearing rules. **Every other skill defers to these when they conflict.** Read `CLAUDE.md`, `docs/PRODUCT_SPEC.md`, `docs/ARCHITECTURE.md`, `docs/DESIGN_SYSTEM.md` before non-trivial work.

## Three invariants (do not violate)

1. **Determinism.** Numbers on the screen must be reproducible from rows in the database. No LLM-generated figures. No silent randomness. No floating-point money — use `Decimal` server-side, `number` with explicit precision client-side via the `Money` component.
2. **Evidence.** Any derived insight that names a cause must return `evidence_ids: list[str]` containing transaction/invoice UUIDs. The frontend resolves them via `EvidenceChip` / `RecordSheet`.
3. **Audit.** Every mutation route calls `record_audit(...)`. Every LLM call is recorded with its facts-payload hash. No exceptions for "internal" tools.

If a feature would violate one of these, change the feature, not the invariant.

## Layered backend (FastAPI)

For any new feature, work in this order — do not skip layers:

1. **Model** (`app/models/<thing>.py`)
   - SQLAlchemy 2 declarative, async session.
   - UUIDs as `PG_UUID(as_uuid=False)` strings with `default=gen_uuid`.
   - Org-scoped tables MUST have `org_id` indexed.
   - Add the new model file to `from app.models import ...` in `alembic/env.py` or autogenerate will silently miss its tables.

2. **Migration** (`alembic revision -m "..." --autogenerate`)
   - Review the autogenerated diff carefully; autogenerate misses index renames and JSONB defaults.
   - Indexes on every FK and every column you `WHERE` on.

3. **Service** (`app/services/<domain>/<file>.py`)
   - **Pure Python.** Inputs: typed primitives, ORM rows, or `dict`. Outputs: dict / dataclass / list. **No I/O inside metric functions.** No sessions, no HTTP, no clocks (pass `today: date` as a parameter; default it at the router layer).
   - This is what makes services unit-testable in milliseconds and provably deterministic.

4. **Pydantic schema** (`app/api/schemas.py`)
   - Request and response models. Use `model_config = ConfigDict(from_attributes=True)` to map from ORM rows.

5. **Router** (`app/api/routers/<thing>.py`)
   - Thin: parse request → fetch rows scoped to `CurrentOrg` → call service → serialize → record audit → publish event (if applicable) → return.
   - Inject `CurrentOrg`, `CurrentUser`, `DbSession` via the aliases in `app/deps.py`.
   - For mutations, call `record_audit(...)`.
   - For state-changing events, call `events.publish(org_id, type, payload)`.

6. **Mount** the router in `app/main.py`.

7. **Tests** (`backend/tests/test_<thing>.py`)
   - Service-level unit tests first (no DB). Use `pytest -k <name>` to iterate.
   - Then a contract test in `tests/test_api_contracts.py` — exercises the route via `TestClient` against an in-memory or scoped DB.

## Layered frontend (Next.js)

For every new feature, plumb:

1. **Type** (`frontend/lib/api/types.ts`) — DTO matching the Pydantic response.
2. **Mapper** (`frontend/lib/api/mappers.ts`) — only if the DTO shape differs from the UI shape. Otherwise skip.
3. **Hook** (`frontend/lib/api/queries/<domain>.ts`) — split per domain; `hooks.ts` is being phased out.
4. **Page** (`frontend/app/(shell)/<thing>/page.tsx`) or component.
5. **Empty / loading / error states** — every one, every time. See `components/common/empty-state.tsx`.

Never call `apiFetch` from a component. Components call hooks; hooks call `apiFetch`.

## Money, dates, money, dates

- Money on the server: `Decimal` everywhere; serialize as `float` only at the router boundary; use `app.utils.money.round_currency`.
- Money on the client: `<Money value={...} currency="USD" />`. Never `toLocaleString` directly.
- Dates on the server: `datetime.date` for calendar dates; `datetime` with `timezone=True` for timestamps. Pass `today` into services; never call `date.today()` inside a service.
- Week boundaries: `app.utils.dates.week_start` (Monday). Don't roll your own.

## LLM features

Every LLM call follows this pattern. There are no exceptions:

```python
from app.services.llm.facts_payload import build_facts_payload
from app.services.llm.guardrails import (
    validate_llm_response, GUARDRAIL_PROMPT_INSTRUCTIONS, build_facts_hash,
)

facts = build_facts_payload(metrics=..., transactions=..., invoices=..., forecast_summary=...)
allowed_ids = {row["id"] for row in facts.get("transactions", []) + facts.get("invoices", [])}

prompt = f"""{GUARDRAIL_PROMPT_INSTRUCTIONS}

Facts:
{json.dumps(facts, default=str)}

User question: {user_q}"""

raw = await openai_call(prompt)
ok, disclaimers, err = validate_llm_response(raw, facts, allowed_ids)
if not ok:
    raise HTTPException(422, detail={"code": "llm_guardrail_violation", "reason": err})

# Persist to llm_calls table with facts_hash for audit
audit.record_llm_call(session, org=org, user=user, facts_hash=build_facts_hash(facts), response=raw)
return {"text": raw, "evidence_ids": list(allowed_ids), "disclaimers": disclaimers}
```

If you're tempted to skip a step "just for prototyping", stop and use the deterministic engine instead.

## RBAC

- Backend is the boundary. Frontend role checks are UX only — never trust them.
- Use the `requires(role="admin")` dependency (build in phase 1 if missing) to gate mutations.
- Read-only endpoints don't need role gating beyond `CurrentOrg`.

## Naming

- Backend snake_case; frontend camelCase. The mapper layer crosses the boundary.
- Service files are named after the noun they operate on (`metrics.py`, `forecast.py`), not the verb (`compute_metrics.py`).
- Routers are named after the URL segment they serve (`spending.py` → `/spending`).
- Tests mirror the file under test (`services/runway/forecast.py` → `tests/test_runway_forecast.py`).

## When in doubt

- **Symptom:** "this endpoint is slow." Check the SQL. We use SQLAlchemy 2 async — eager-load relationships with `selectinload`, never N+1.
- **Symptom:** "I need to compute a number in the LLM response." Don't. Move the computation server-side, put the resulting number in the facts payload, let the LLM quote it.
- **Symptom:** "I need an admin-only field on a list endpoint." Branch on the role and omit the field for non-admins; do not return then filter on the client.
- **Symptom:** "I want to ship this without a test." Re-read the project invariants above.
