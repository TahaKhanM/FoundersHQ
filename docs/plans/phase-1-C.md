# Phase 1.C — Notifications Inbox

> **For agentic workers:** REQUIRED SUB-SKILL: `superpowers:subagent-driven-development` or `superpowers:executing-plans`. Steps use `- [ ]`.

**Goal:** unread-count bell in the top bar; a `/(shell)/inbox` page that lists, marks read, archives, and snoozes notifications; per-type preferences; SSE-driven live updates via the `lib/realtime.ts` client.

**Architecture:** the `Notification` model already exists. We add a `NotificationPreference` table (per user, per type) and wire a frontend page + bell component. SSE channel `notification.created` plumbed via 0.B's events publisher.

**Tech Stack:** existing `notifications` router; new pref-store endpoints; `lib/realtime.ts` from 0.C; shadcn primitives.

**Dependencies:** 1.A merged (RBAC + audit helper). Independent of 1.B/1.D/1.E/1.F.

**Skills:** `foundershq-conventions`, `superpowers:test-driven-development`, `realtime-and-streaming`, `vercel:nextjs`.

## Tasks

1. **Schema** — `snoozed_until` on `notifications`; `notification_preferences` table (`user_id`, `type`, `email: bool`, `in_app: bool`, `created_at`). Migration via autogenerate. List endpoint filters out items where `snoozed_until > now()` from Unread.
2. **Endpoints** — snooze, prefs get/put. All mutations: `record_audit` + `publish_event("notification.updated", payload={"id": …, "status": "archived"})` etc.
3. **SSE event on create** — when an existing notification generator creates a new row (today: only `app/tasks/jobs.py` writes them), call `publish_event("notification.created", payload=NotificationDTO)`. Add this to the one place that creates notifications today.
4. **Bell component** — uses `useNotificationCount` hook; opens a popover with the 5 most recent unread items; "See all" link to `/inbox`.
5. **Inbox page** — three tabs as URL search params (`/inbox?tab=unread`). Empty state with CTA "Connect a bank to start receiving insights" (deep-link to `/integrations`).
6. **Preferences UI** — table of types × channels; toggles persisted on change with optimistic update.
7. **Realtime subscription** — `useRealtimeChannel("notification.created", invalidate)` revalidates the SWR cache so new items appear instantly.

## Tests

- `test_notifications_inbox.py` — snooze hides item until `snoozed_until`; prefs CRUD; audit rows; `publish_event` called.
- Playwright: drive a synthetic notification publish via a test endpoint, see the bell badge increment and the inbox list update without a refresh.

## Hard rules

- No new dependencies (`recharts`/`lucide-react`/`swr` already cover us).
- `publish_event` must not block the DB commit on Redis failures — wrap in try/except.
- Bell shows `99+` for >99 unread (don't render four-digit numbers).
- Snooze options: 1h, 4h, 24h, until next Monday — discrete; no free-form datetime picker in phase 1.

## Definition of done

- `make verify` + `pnpm verify` clean.
- Notification rows update live in `pnpm dev` (tested via a manual publish or test endpoint).
- `<Bell />` shows correct unread badge; popover lists 5 most recent unread.
- `/inbox` renders three tabs, archive + snooze + mark-read all work.
- Preferences persist across reload.
